# No Testing Policy

Do not create, write, modify, or run any test cases or test files for this project.

This includes:
- Writing new unit, integration, or end-to-end tests.
- Modifying existing test files.
- Running test suites or test commands (e.g. `npm test`, `vitest`, `jest`).

If verification of a change is needed, verify by reading code, running the build/typecheck, or manual reasoning instead of tests. Do not add test coverage even if it seems standard practice, unless the user explicitly asks for it again in the future.
